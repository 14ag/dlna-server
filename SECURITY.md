# Security Policy

## Supported Versions

Currently, only the latest release is actively supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a vulnerability in WinDLNAServer, please report it by opening an issue on GitHub, taking care to check the "Security Vulnerability" option if the platform provides one, or send an email if a maintainer has provided one in the repository settings. 

We try to acknowledge all security reports within 3-5 business days. We will work to patch serious flaws in networking (SSDP, HTTP layer) as a priority. If your vulnerability report is accepted, a CVE may be requested, and we will credit you in the release notes.
